CSS Specificity Exercise

Live demonstration of CSS specificity rules from Lab 4 — Exercise 1. Each paragraph below is styled by a different selector combination.

Live Demo

1. Plain <p> — element selector → blue (0-0-1)

2. <p class="text"> — class selector wins → green (0-1-0)

3. Same but p.text rule applies → red (0-1-1)

4. id="unique" — ID selector wins → purple (1-0-0)

5. class="text highlight" — two classes → orange (0-2-1)

Answers & Explanation

What colour does each paragraph render, and why?

  1. Plain <p>: blue — only p (0-0-1) applies.
  2. <p class="text">: green — .text (0-1-0) beats p (0-0-1).
  3. <p class="text"> with p.text: red — (0-1-1) beats (0-1-0).
  4. <p id="unique">: purple — ID selector (1-0-0) beats all class/element combos.
  5. <p class="text highlight">: orange — p.text.highlight (0-2-1) beats p.text (0-1-1).

Key rule: Specificity = (inline, ID, class, element). Higher specificity always wins regardless of order in the stylesheet.
